# Veloadout — Requirements

## Product vision

Веб-приложение для байкпакеров: пользователь вводит снаряжение, система считает суммарный объём и рекомендует нужные сумки. Если товар не найден в базе — Claude AI ищет его в интернете и возвращает реальные данные.

---

## Functional requirements

### FR-1: Поиск снаряжения

| # | Требование | Статус |
|---|---|---|
| FR-1.1 | Пользователь вводит название товара в поисковую строку | ✅ |
| FR-1.2 | Быстрый поиск в базе данных (без AI) | ✅ |
| FR-1.3 | Если не найдено — AI ищет в интернете через Claude Haiku + web_search | ✅ |
| FR-1.4 | AI-результат показывается как карточка подтверждения, не добавляется автоматически | ✅ |
| FR-1.5 | Для товаров с несколькими размерами AI возвращает все варианты с объёмом и весом | ✅ |
| FR-1.6 | Пользователь выбирает размер (Regular / Long / Wide и т.д.) перед добавлением | ✅ |
| FR-1.7 | Если в запросе указан размер — нужный вариант выделяется автоматически | ✅ |
| FR-1.8 | Подтверждённый AI-товар сохраняется в общий каталог (для всех пользователей) | ✅ |
| FR-1.9 | При повторном запросе товар находится из БД, варианты показываются снова | ✅ |

## Non-functional requirements

### NFR-1: Производительность

| # | Требование | Решение |
|---|---|---|
| NFR-1.1 | Поиск в БД — быстрый без AI-задержки | Two-stage search: db_only → AI |
| NFR-1.2 | Поиск по каталогу масштабируется при росте базы | GIN trigram index на `search_text` |
| NFR-1.3 | AI-поиск — только когда товар не найден в БД | `db_only=1` fast path |

### NFR-2: Безопасность

| # | Требование | Решение |
|---|---|---|
| NFR-2.1 | Rate limiting на AI-поиск (дорогой ресурс) | 20 запросов / IP / час |
| NFR-2.2 | Rate limiting на magic link (анти-спам) | 5 req / IP + 3 req / email / 10 мин |
| NFR-2.3 | Валидация всех входных данных | Zod на всех API роутах |
| NFR-2.4 | Security headers | HSTS, X-Frame-Options, Referrer-Policy, Permissions-Policy |
| NFR-2.5 | Нет открытых debug-эндпоинтов | `/api/debug-env` удалён |
| NFR-2.6 | Данные пользователей изолированы | Supabase RLS |

### NFR-3: Надёжность

| # | Требование | Решение |
|---|---|---|
| NFR-3.1 | Ошибки сохранения видны пользователю | Toast-уведомления |
| NFR-3.2 | Нет race condition при автосохранении | AbortController отменяет предыдущий запрос |
| NFR-3.3 | Ошибки рендера не крашат весь UI | Error boundary + global-error |
| NFR-3.4 | 404 страница | Кастомный not-found |
| NFR-3.5 | Тесты на domain-логику | 11 Vitest тестов |

### NFR-4: GDPR / Правовое

| # | Требование | Решение |

## AI behaviour requirements

| # | Требование |
|---|---|
| AI-1 | Использовать **сжатый/упакованный** объём, не распакованный |
| AI-2 | Реалистичные диапазоны: палатки 3–12L, спальники 2–8L, пуховики 1–4L |
| AI-3 | Для товаров с размерами — возвращать массив `variants` со всеми размерами |
| AI-4 | Multi-turn web_search loop (до 5 итераций) |
| AI-5 | Fallback на knowledge-only если web_search недоступен |
| AI-6 | Confidence: `high` / `medium` / `low` |
| AI-7 | `volumeNote` с пояснением (сжатый, компрессионный мешок и т.д.) |
| AI-8 | Имена на всех трёх языках (EN / DE / RU) |
